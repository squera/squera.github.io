Advanced Programming 2022-23

By prof. Marco Patrignani

General Course Information

This Advanced Programming course has two goals: (1) making you well-versed in a novel, advanced programming language, and (2), making you capable of developing large projects without relying solely on your code. Thus, the course will first present the Rust programming language, and then it will have a major W3C-style class-wide project.

The first part of the class will be more Rust-focussed, while the latter one will be more project-focussed, with working group meetings and the software fair. Time permitting (their time), we will have speakers from the industry come in and share their Rust experience with you.

High-level Syllabus

What is Rust, and Why Rust?

Rust is a relatively new programming language that promises both type (and memory) safety as well as the speed and expressiveness of C++. Rust is multi-paradigm, it is especially well suited for systems programming, and it is gaining enormous traction in industry – virtually all major software companies use Rust. Expected topics covered include: the Rust type system (structs, enums, traits, generics), memory management (safety, ownership, borrowing, and lifetimes), multiparadigm programming (functional –closures, higher order functions – and object-oriented – traits, polymorphism –). Note that the goal of class is not to explore all of the nitty-gritty advanced features of Rust (e.g., concurrency or the full trait system), but you’ll have enough of an expertise to delve into those topics afterwards!

What is a W3C-style class-wide project and Why should we do it?

Software is not developed in isolation. Even within the same companies, the final product is the outcome of the interoperation of bits of code written by different company teams. What is crucial is that these teams use each other’s code, and often also external code, according to some documented API, or specification (aka spec). It is therefore important for real-world programmers to be able to not just rely on their code, but be able to both use other people’s code as well as supply other people with working code. But what is working code? For the sake of simplicity, we’ll assume that working code is code that behaves as an agreed-upon specification. And here is where the W3C aspect of class kicks in. With a gross oversimplification, W3C is the main international standards organization that develops those protocols that make the World Wide Web work. In a smaller scale, the class will assume a W3C-like role (as well as structure) and develop the spec that will make the project work. Thus, a part of the project will be agreeing on a spec, implement it and ‘sell’ it. Then, another part will be writing code that uses (possibly multiple versions of) that implementation, all of which are not developed by your group.
